# L1: NixOS — The Same Concepts, A Richer Module Set

> **Goal**: Understand what NixOS is and how it differs from nix-darwin. Add a NixOS configuration to your existing flake. Recognize that everything you know already applies.

## When to Start This Lesson

Start this when you have homelab hardware (or a VM) ready to run NixOS.

## Prerequisites

- D2 complete — modular config structure in place
- A machine (physical or VM) to install NixOS on

---

## Concepts

### NixOS is nix-darwin for Linux, But Deeper

With nix-darwin, Nix manages *some* of your Mac — packages, preferences, services. macOS itself is still the base.

With NixOS, Nix manages *everything*:
- The kernel and boot loader
- All system services (systemd units)
- Network configuration
- User accounts and SSH keys
- Filesystem layout
- Every installed package

There is no "base OS" beneath NixOS that you configure manually. The entire system is declared in Nix.

```
Terraform analogy:
  nix-darwin    → Terraform managing some AWS resources, but EC2 is pre-configured
  NixOS         → Terraform managing everything including the base AMI construction
```

### The Same Module System

NixOS uses the exact same module system as nix-darwin. The same `imports`, `options`, `config`, `specialArgs` patterns apply. The difference is just the available options.

```
nix-darwin option:        system.defaults.dock.autohide = true;
NixOS equivalent:         services.xserver.desktopManager.gnome.enable = true;

nix-darwin option:        services.nix-daemon.enable = true;
NixOS option:             nix.settings.experimental-features = [ "nix-command" "flakes" ];

Both use:                 environment.systemPackages
                          home-manager.users.<name>
                          nix.settings.*
                          nixpkgs.*
```

### `nixpkgs.lib.nixosSystem` vs `nix-darwin.lib.darwinSystem`

The flake entry point changes from `darwinSystem` to `nixosSystem`:

```nix
# In flake.nix:
nixosConfigurations."homelab" = nixpkgs.lib.nixosSystem {
  system = "x86_64-linux";    # or aarch64-linux for ARM
  modules = [
    ./hosts/homelab/default.nix
    home-manager.nixosModules.home-manager    # note: nixosModules, not darwinModules
  ];
  specialArgs = {
    username = "kacper";
    hostname = "homelab";
  };
};
```

### `hardware-configuration.nix` — Auto-Generated

NixOS generates a `hardware-configuration.nix` during installation that describes your specific hardware:
- Which disks to use and how they're partitioned
- Which kernel modules to load
- What filesystems exist

You import this file in your host module. You don't write it manually.

```nix
# hosts/homelab/default.nix
{ username, ... }:
{
  imports = [
    ./hardware-configuration.nix    # auto-generated by NixOS installer
    ../../modules/shared/packages.nix    # reused from macOS config!
    ../../modules/nixos/services.nix
  ];

  # Different options from nix-darwin, same syntax:
  boot.loader.grub.enable = true;
  boot.loader.grub.device = "/dev/sda";

  networking.hostName = "homelab";
  time.timeZone = "Europe/Warsaw";

  users.users.kacper = {
    isNormalUser = true;
    extraGroups = [ "wheel" "docker" ];
    openssh.authorizedKeys.keys = [
      "ssh-ed25519 AAAA..."    # your public key
    ];
  };

  services.openssh.enable = true;

  nixpkgs.hostPlatform = "x86_64-linux";
  system.stateVersion = "24.11";
}
```

### The Reuse Payoff

Notice: `../../modules/shared/packages.nix` is the same file you wrote for macOS. It works on NixOS unchanged because it only uses `environment.systemPackages` and `pkgs` — both available on both platforms.

This is the reward for the `shared/` vs `darwin/` module split from D2.

## Exercise 1: Install NixOS

**On a VM** (recommended for first time):
```bash
# Download the NixOS ISO from https://nixos.org/download
# Boot from it
# Follow the installer or use the manual partitioning approach
```

**During installation**, NixOS generates `hardware-configuration.nix`. Copy this file to your nix-config repo.

## Exercise 2: Add NixOS to your flake

```nix
# flake.nix — add nixosConfigurations alongside darwinConfigurations:
outputs = { self, nixpkgs, nix-darwin, nix-homebrew, home-manager }: {

  darwinConfigurations."kacpers-MacBook-Pro" = nix-darwin.lib.darwinSystem { ... };

  nixosConfigurations."homelab" = nixpkgs.lib.nixosSystem {
    system = "x86_64-linux";
    modules = [
      ./hosts/homelab/default.nix
      home-manager.nixosModules.home-manager
    ];
    specialArgs = {
      username = "kacper";
      hostname = "homelab";
    };
  };
};
```

## Exercise 3: Create `hosts/homelab/default.nix`

Write the host module using the template from the Concepts section above. Import your `hardware-configuration.nix` and the shared modules.

## Exercise 4: First NixOS rebuild

```bash
# On the NixOS machine:
nixos-rebuild switch --flake github:yourusername/nix-config#homelab
# or if the repo is local:
nixos-rebuild switch --flake /path/to/nix-config#homelab
```

## Comprehension Questions

1. **What does `hardware-configuration.nix` contain?** Can you write it manually, or must you generate it?

2. **What changes between `darwinSystem` and `nixosSystem` in your flake?** What stays the same?

3. **Which of your existing modules can be reused on NixOS without modification?** Which ones are macOS-specific?

4. **On NixOS, there's no macOS. What manages the boot loader, kernel, and init system?** How are these declared?

5. **If you add a new CLI tool to `modules/shared/packages.nix`**, when does it appear on: your Mac (after what command)? your NixOS server (after what command)?
